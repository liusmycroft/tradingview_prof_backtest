# 因子策略组合报告

## 1. 组合原则

本报告基于 `src/factor_type_classification_report.md` 的风格分类结果，尝试把不同风格的因子拼成几套更接近可交易策略的组合框架。

组合时遵循四个原则：

- 不做“同类因子堆叠”，尽量跨风格搭配，降低单一市场状态失效的风险。
- 优先使用 `FEASIBLE` 和 `PARTIAL` 因子，把 `NOT_FEASIBLE` 因子视作未来可扩展增强项。
- 每套策略至少包含四层角色：方向因子、确认因子、风险过滤因子、执行/流动性过滤因子。
- 先追求逻辑闭环，再追求因子数量；宁可少而互补，不要多而重复。

## 2. 建议的组合结构模板

一个更稳的组合，一般可以拆成：

1. 核心方向层：决定做多、做空还是观望。
2. 风格确认层：确认这次信号是否真有共振。
3. 风险过滤层：避免在错误的波动/拥挤状态下入场。
4. 交易执行层：确保信号不是由流动性扭曲出来的假象。

## 3. 具体策略方案

### 策略一：趋势确认突破

#### 适用场景

适合趋势行情、板块主升、个股走出中短期强势通道的环境。

#### 推荐因子组合

- 趋势方向：`TREND_CLARITY`, `TREND_RATIO`
- 资金确认：`TREND_CAPITAL_NET_SUPPORT`, `SMART_MONEY`
- 量能确认：`PATV`, `ABNVOLD`
- 执行过滤：`PRICE_RESILIENCY`, `LIQUIDITY_PREMIUM_EW`

#### 为什么这样组合

- `TREND_CLARITY` 和 `TREND_RATIO` 解决的是“是不是趋势行情”。
- `TREND_CAPITAL_NET_SUPPORT` 和 `SMART_MONEY` 解决的是“这段趋势背后有没有更强资金在支持”。
- `PATV` 和 `ABNVOLD` 解决的是“量能是不是持续跟得上，而不是一天脉冲后就熄火”。
- `PRICE_RESILIENCY` 和 `LIQUIDITY_PREMIUM_EW` 解决的是“这个趋势能不能以较小冲击成本继续交易”，避免买在纯流动性挤压的末端。

#### 组合逻辑

当趋势清晰、价格位移效率高、趋势资金净支撑为正、聪明钱活跃、异常成交量持续，同时流动性没有显著恶化时，趋势延续成功率通常更高。

#### 风险点

- 横盘假突破时容易失效。
- 市场从趋势切换到震荡时，资金确认层往往会滞后。

### 策略二：高关注过度反应后的均值回归

#### 适用场景

适合公告刺激、涨跌停事件、情绪过热或单日异常收益过大的环境。

#### 推荐因子组合

- 注意力触发：`ATTENTION_ABNORMAL_RETURN`, `ATTENTION_EVENTS`, `DAILY_RETURN_ATTENTION`
- 反转核心：`AB_PR`, `AB_NR`, `CORRECTED_INTRADAY_REVERSAL`
- 情绪放大：`VOL_PANIC`, `SALIENCE_RETURN`
- 质量过滤：`SNR_ENHANCED_REVERSAL`

#### 为什么这样组合

- 注意力类因子负责识别“市场是不是都看到了这只股票”。
- `AB_PR`、`AB_NR`、`CORRECTED_INTRADAY_REVERSAL` 负责识别“关注之后，价格是否已经走到短期失衡区”。
- `VOL_PANIC` 和 `SALIENCE_RETURN` 是典型的过度反应放大器，越惊恐、越凸显，越容易形成短期偏离。
- `SNR_ENHANCED_REVERSAL` 是关键过滤器，它把低质量的噪声反转剔掉，尽量只保留“有结构、有可逆性”的反转。

#### 组合逻辑

先让注意力因子筛出热点和异动股，再让反转因子判断价格是否过冲，最后用情绪和信号质量做双重确认。这种组合比单纯做反转更稳，因为它不是“看到跌就抄底”，而是“看到被过度关注后的错误定价再反向交易”。

#### 风险点

- 强趋势龙头可能出现“高关注但不回头”。
- 在极强主题行情里，注意力上升可能意味着继续加速而不是回归。

### 策略三：恐慌出清后的修复反弹

#### 适用场景

适合隔夜利空、日内深跌、尾盘恐慌抛售后出现修复信号的环境。

#### 推荐因子组合

- 冲击识别：`OVERNIGHT_GAP`, `INTRADAY_MAX_DRAWDOWN`, `RVJN`
- 超跌确认：`WEIGHTED_LOWER_SHADOW_FREQ`, `VSA_CLOSE_DIFF`
- 资金回流：`LCVOL`, `NET_SUPPORT_VOLUME`
- 额外过滤：`HF_DOWNSIDE_VOL_RATIO`, `VCVaR`

#### 为什么这样组合

- `OVERNIGHT_GAP`、`INTRADAY_MAX_DRAWDOWN`、`RVJN` 用来确定“跌得是否足够急、足够深、足够像情绪性冲击”。
- `WEIGHTED_LOWER_SHADOW_FREQ` 和 `VSA_CLOSE_DIFF` 则用于判断“低位是否开始出现承接和下影线式止跌”。
- `LCVOL` 与 `NET_SUPPORT_VOLUME` 用来确认“卖压释放后是否有增量资金接回去”。
- `HF_DOWNSIDE_VOL_RATIO` 和 `VCVaR` 是风险闸门，防止在真正的趋势性崩塌里贸然做修复。

#### 组合逻辑

只有当冲击已经发生、超跌结构出现、低位承接成立、回流资金开始补位时，才去参与修复。这样可以把“抄底”从纯主观动作，变成“冲击完成 + 承接出现 + 资金回流”的结构化交易。

#### 风险点

- 对单边下跌趋势的第二段杀跌仍然敏感。
- 需要配合严格止损，避免把反弹交易做成趋势逆势交易。

### 策略四：低波稳态的中期持有

#### 适用场景

适合震荡偏上、机构偏好稳健资产、市场风险偏好不高但结构性机会存在的环境。

#### 推荐因子组合

- 波动约束：`SIMILAR_LOW_VOL`, `IDEAL_AMPLITUDE`, `DOWNSIDE_RV`
- 趋势匹配：`TREND_CLARITY_MOMENTUM`
- 拥挤过滤：`CSAD_SMALL`, `CSSD`
- 执行过滤：`LIQUIDITY_PREMIUM_EW`, `SHADOW_CLOSE_STD`

#### 为什么这样组合

- `SIMILAR_LOW_VOL`、`IDEAL_AMPLITUDE`、`DOWNSIDE_RV` 用来定义“价格波动是否处于可持有区间”。
- `TREND_CLARITY_MOMENTUM` 让组合不只是买低波，而是买“低波且趋势结构匹配”的资产。
- `CSAD_SMALL` 和 `CSSD` 负责过滤掉局部抱团和羊群过热。
- `LIQUIDITY_PREMIUM_EW` 与 `SHADOW_CLOSE_STD` 则保证标的不是表面低波、实则流动性差或多空博弈剧烈。

#### 组合逻辑

这套组合本质上不是追求最强爆发，而是追求“低波、清晰、可持有、不过度拥挤”的稳态收益。它和前面几套更偏短周期、事件驱动的策略形成明显互补。

#### 风险点

- 在极强风险偏好行情里可能显著跑输高弹性组合。
- 如果低波来自流动性匮乏而不是真稳健，会造成错配。

### 策略五：行业扩散与主力跟随

#### 适用场景

适合主题扩散、行业轮动和板块内部强弱传导明显的环境。

#### 推荐因子组合

- 联动源头：`INDUSTRY_MOMENTUM`, `RNBR_TOV`, `CO_MIN`
- 价格确认：`PV_CORR_TREND`, `LARGE_VOL_PRICE_CORR`
- 资金确认：`MAIN_FORCE_STRENGTH`, `MORNING_INFLOW_STABILITY`
- 注意力辅助：`ATTENTION_MARKET_VOLUME`

#### 为什么这样组合

- `INDUSTRY_MOMENTUM`、`RNBR_TOV`、`CO_MIN` 用来确认收益和换手是否正在从板块核心向周边扩散。
- `PV_CORR_TREND` 和 `LARGE_VOL_PRICE_CORR` 用来判断这种扩散是不是被真实交易推动，而不是纯消息刺激。
- `MAIN_FORCE_STRENGTH` 和 `MORNING_INFLOW_STABILITY` 让策略尽量站在主力资金同一侧。
- `ATTENTION_MARKET_VOLUME` 可以帮助区分“板块内部自然扩散”与“市场全面噪声抬升”。

#### 组合逻辑

先找板块级联动，再看个股级量价确认，最后用主力资金稳定性做过滤。这类策略的核心不是买最强龙头，而是买“刚被扩散到、且资金开始接力”的第二梯队。

#### 风险点

- 板块轮动速度太快时，扩散链条可能来不及完成。
- 容易在板块尾声接到被动跟涨股。

### 策略六：大单驱动的机构累积

#### 适用场景

适合中等流动性以上、主力资金有连续建仓痕迹、价格尚未完全加速的环境。

#### 推荐因子组合

- 大单信号：`IMPROVED_LARGE_RATIO`, `LARGE_ORDER_RETURN`
- 主力活跃：`MAIN_FORCE_STRENGTH`, `MTE`
- 资金稳定性：`MORNING_INFLOW_STABILITY`, `TREND_CAPITAL_VWAP`
- 成本约束：`PRICE_IMPACT_BIAS`, `PRSI`

#### 为什么这样组合

- `IMPROVED_LARGE_RATIO` 与 `LARGE_ORDER_RETURN` 用来确定“大单是不是在真实推价格，而不是无效对倒”。
- `MAIN_FORCE_STRENGTH` 与 `MTE` 用来确认这些大单是不是带有更一致的主导方向。
- `MORNING_INFLOW_STABILITY` 与 `TREND_CAPITAL_VWAP` 则用于识别“是不是连续、稳定、低噪声地在吸筹”。
- `PRICE_IMPACT_BIAS` 与 `PRSI` 用来控制建仓成本与冲击失真，避免买入只是因为盘口太薄。

#### 组合逻辑

这是一套更偏机构交易画像的策略。它关注的不是“今天强不强”，而是“是否有持续性的、相对隐蔽但可被交易痕迹观察到的建仓过程”。

#### 风险点

- 需要较稳定的分钟级或近似分钟级数据支持。
- 对流动性突变比较敏感。

## 4. 这几套策略之间为什么能互补

### 4.1 方向来源不同

- 策略一来自趋势延续
- 策略二来自注意力过热后的回归
- 策略三来自恐慌后的修复
- 策略四来自低波稳态
- 策略五来自行业扩散
- 策略六来自机构累积

它们依赖的市场机制不同，因此天然有分散化价值。

### 4.2 对市场状态的敏感性不同

- 趋势市中，策略一和策略五通常更占优。
- 事件冲击或情绪失衡时，策略二和策略三更有效。
- 风格偏防守时，策略四通常更稳。
- 主力隐性吸筹阶段，策略六往往更早给出信号。

### 4.3 风险暴露不同

- 趋势类更怕震荡。
- 反转类更怕单边。
- 低波类更怕风险偏好突然上冲。
- 扩散类更怕主题切换过快。
- 机构累积类更怕流动性错判。

正因为失败模式不同，把它们做成组合往往比单独使用更稳。

## 5. 如果只做当前可落地版本，优先顺序建议

若以当前仓库的数据条件和实现优先级来排序，我建议：

1. 先做 `策略一：趋势确认突破`
2. 再做 `策略二：高关注过度反应后的均值回归`
3. 然后做 `策略三：恐慌出清后的修复反弹`
4. 最后再扩展 `策略五` 和 `策略六`

原因是：

- 策略一和策略二最容易形成清晰、可解释、可验证的信号闭环。
- 策略三有较好的实战意义，但风险过滤必须更严格。
- 策略五和策略六对分钟级质量、联动口径和交易细节更敏感，适合作为第二阶段增强。

## 6. 简短结论

从这份因子库出发，最值得优先搭建的，不是“万能大杂烩策略”，而是几套**市场机制不同、风险来源不同、风格彼此互补**的策略框架。

更具体地说：

- 用 `趋势 + 资金 + 流动性` 做延续
- 用 `注意力 + 反转 + 情绪` 做回归
- 用 `冲击 + 承接 + 资金回流` 做修复
- 用 `低波 + 清晰趋势 + 去拥挤` 做稳态持有
- 用 `行业扩散 + 主力确认` 做板块接力
- 用 `大单推进 + 成本约束` 做机构累积

这样组合的好处，是你拿到的不是一堆孤立因子，而是一套可以继续向回测框架、打分模型和实盘筛选器推进的“策略积木”。
